# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

В приложении используются две сущности, описывающие данные - **товар** и **покупатель**.

Товар:

```ts
interface IProduct {
    id: string;
    title: string;
    image: string;
    category: string;
    price: number | null;
    description: string;
}
```

Интерфейс `IProduct` описывает товар, который можно купить в магазине:
`id` - уникальный идентификатор товара;  
`title` - название товара;  
`image` - ссылка на изображение;  
`category` - категория товара;  
`price` - цена товара; может быть `null`, если товар бесценен;  
`description` - подробное текстовое описание товара;

Покупатель:

```ts
interface IBuyer {
    payment: TPayment;
    address: string;
    email: string;
    phone: string;
}
```

Интерфейс `IBuyer` описывает данные, которые покупатель указывает при оформлении заказа:
`payment` - выбранный способ оплаты;  
`address` - адрес доставки;  
`email` - электронная почта;  
`phone` - номер телефона;

Способ оплаты заказа описывается отдельным типом:

```ts
type TPayment = 'card' | 'cash';
```

`TPayment` - допустимые значения способа оплаты, заданные функциональными требованиями: онлайн-оплата картой (`card`) или оплата при получении (`cash`)

Для описания результата валидации данных покупателя используется вспомогательный тип:

```ts
type TBuyerErrors = Partial<Record<keyof IBuyer, string>>;
```

`TBuyerErrors` - объект, в котором ключами выступают поля `IBuyer`, а значениями - текст ошибки. Если поле валидно, соответствующего ключа в объекте нет.

### Типы обмена с сервером

Чтобы не описывать данные дважды, типы для обмена с сервером строятся на основе уже описанных интерфейсов.

Ответ сервера на запрос каталога (`GET /product/`):

```ts
interface IProductListResponse {
    total: number;
    items: IProduct[];
}
```

`total` - общее количество товаров; `items` - массив товаров.

Данные заказа, отправляемые на сервер (`POST /order/`). Объединяют данные покупателя, итоговую сумму и список идентификаторов выбранных товаров:

```ts
interface IOrderRequest extends IBuyer {
    total: number;
    items: string[];
}
```

`total` - итоговая стоимость заказа; `items` - массив `id` товаров из корзины.

Ответ сервера на оформление заказа (`POST /order/`):

```ts
interface IOrderResponse {
    id: string;
    total: number;
}
```

`id` - идентификатор созданного заказа; `total` - сумма, на которую подтверждён заказ.

## Модели данных

### Класс ProductsModel

Отвечает за хранение каталога всех доступных товаров и за хранение товара, выбранного пользователем для подробного отображения.

Конструктор класса не принимает параметров, инициализирует пустой массив товаров и отсутствие выбранного товара.

Поля класса:  
`_items: IProduct[]` - массив всех товаров, доступных в магазине.  
`_selectedItem: IProduct | null` - товар, выбранный для подробного отображения; `null`, если ничего не выбрано.

Методы класса:  
`setItems(items: IProduct[]): void` - сохраняет массив товаров, полученный в параметре, в поле `_items`.  
`getItems(): IProduct[]` - возвращает массив всех товаров из модели.  
`getItem(id: string): IProduct | undefined` - возвращает один товар по его `id`; `undefined`, если товар не найден.  
`setSelectedItem(item: IProduct): void` - сохраняет переданный товар в поле `_selectedItem` для подробного отображения.  
`getSelectedItem(): IProduct | null` - возвращает товар, выбранный для подробного отображения.

### Класс BasketModel

Отвечает за хранение товаров, выбранных покупателем для покупки.

Конструктор класса не принимает параметров, инициализирует пустой массив товаров корзины.

Поля класса:  
`_items: IProduct[]` - массив товаров, добавленных покупателем в корзину.

Методы класса:  
`getItems(): IProduct[]` - возвращает массив товаров, которые находятся в корзине.  
`addItem(item: IProduct): void` - добавляет переданный товар в массив корзины.  
`removeItem(item: IProduct): void` - удаляет переданный товар из массива корзины по его `id`.  
`clear(): void` - очищает корзину, удаляя все товары.  
`getTotalPrice(): number` - возвращает суммарную стоимость всех товаров в корзине (товары с ценой `null` в расчёт не берутся).  
`getCount(): number` - возвращает количество товаров в корзине.  
`hasItem(id: string): boolean` - проверяет наличие товара в корзине по его `id`; возвращает `true`, если товар присутствует.

### Класс BuyerModel

Отвечает за хранение данных покупателя, которые он указывает при оформлении заказа, а также за их валидацию.

Конструктор класса не принимает параметров, инициализирует поля пустыми значениями.

Поля класса:  
`_payment: TPayment | ''` - выбранный способ оплаты; пустая строка, пока способ не выбран.  
`_address: string` - адрес доставки.  
`_email: string` - электронная почта.  
`_phone: string` - номер телефона.

Методы класса:  
`setData(data: Partial<IBuyer>): void` - сохраняет в модель переданные поля. Принимает частичный объект `IBuyer`, поэтому можно записать только одно значение (например, только адрес), не затирая остальные уже сохранённые поля.  
`getData(): IBuyer` - возвращает все данные покупателя одним объектом.  
`clear(): void` - очищает все данные покупателя, сбрасывая поля к пустым значениям.  
`validate(): TBuyerErrors` - проверяет данные покупателя. Поле считается валидным, если оно не пустое. Возвращает объект с ошибками: ключ - невалидное поле, значение - текст ошибки. Для валидных полей ключ в объекте отсутствует; если объект пуст, все данные корректны.

## Слой коммуникации

### Класс WebLarekApi

Отвечает за взаимодействие приложения с API сервера «Web-ларёк»: получение каталога товаров и отправку оформленного заказа. Использует композицию - не наследует базовый класс `Api`, а получает готовый экземпляр через конструктор и вызывает его методы `get` и `post`.

Конструктор:  
`constructor(api: IApi)` - принимает объект, соответствующий интерфейсу `IApi`, и сохраняет его для дальнейших запросов.

Поля класса:  
`_api: IApi` - экземпляр клиента API, через который выполняются запросы к серверу.

Методы класса:  
`getProducts(): Promise<IProductListResponse>` - выполняет `GET` запрос на эндпоинт `/product/` и возвращает промис с объектом, содержащим массив товаров.  
`orderProducts(order: IOrderRequest): Promise<IOrderResponse>` - выполняет `POST` запрос на эндпоинт `/order/`, передаёт в теле данные заказа, полученные в параметре, и возвращает промис с объектом, подтверждающим покупку и её сумму.

## Слой представления (View)

Слой представления отвечает за отображение данных в интерфейсе и за генерацию событий в ответ на действия пользователя. Каждый класс представления отвечает за свой блок разметки на странице.
Представления не хранят данные и не содержат бизнес-логики - они только отображают переданные данные через сеттеры и сообщают о действиях пользователя с помощью событий, которые обрабатывает Презентер.

### Класс Header

Отвечает за отображение шапки сайта - кнопки корзины со счётчиком товаров.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` - принимает брокер событий и корневой DOM-элемент шапки.

Поля класса:  
`counterElement: HTMLElement` - элемент счётчика товаров в корзине (`.header__basket-counter`).  
`basketButtonElement: HTMLButtonElement` - кнопка корзины (`.header__basket`).  
`events: IEvents` - брокер событий.

Методы класса:  
`set counter(value: number)` - устанавливает текущее значение товаров в корзине.

При клике на кнопку корзины генерируется событие `basket:open`.

### Класс Gallery

Отвечает за отображение каталога карточек товаров на главной странице. Не генерирует событий - только отображает переданные ему готовые DOM-элементы карточек.

Конструктор:  
`constructor(container: HTMLElement)` - принимает корневой DOM-элемент каталога.

Класс не хранит собственных полей помимо контейнера.

Методы класса:  
`set items(value: HTMLElement[])` - заполняет контейнер каталога переданным массивом DOM-элементов карточек товаров.

### Класс Modal

Отвечает за отображение модального окна. Не имеет дочерних классов - от него не наследуются другие классы. Всё содержимое, которое отображается внутри модального окна (карточка товара, корзина, формы заказа и контактов, сообщение об успехе), реализовано как самостоятельные независимые компоненты, не связанные с `Modal` отношением наследования. Это позволяет использовать эти компоненты и в любом другом месте интерфейса, не только в модальном окне.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` - принимает брокер событий и корневой DOM-элемент модального окна.

Поля класса:  
`closeButton: HTMLButtonElement` - кнопка закрытия модального окна.  
`content: HTMLElement` - контейнер содержимого модального окна.  
`events: IEvents` - брокер событий.

Методы класса:  
`open(): void` - открывает модальное окно, генерирует событие `modal:open`.  
`close(): void` - закрывает модальное окно и очищает его содержимое, генерирует событие `modal:close`.

### Класс Basket

Отвечает за отображение содержимого корзины. Самостоятельный компонент, не зависящий от `Modal`.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` - принимает брокер событий для обработки клика на кнопку оформления заказа и корневой DOM-элемент корзины.

Поля класса:  
`list: HTMLElement` - контейнер списка товаров.  
`price: HTMLElement` - элемент суммы заказа.  
`button: HTMLButtonElement` - кнопка перехода к оформлению заказа.  
`events: IEvents` - брокер событий.

Методы класса:  
`set items(value: HTMLElement[])` - отображает переданный массив DOM-элементов товаров в списке корзины; если массив пуст, выводит заглушку (например, текст «Корзина пуста») и блокирует кнопку оформления.  
`set price(value: number)` - устанавливает итоговую сумму заказа.  
`set buttonDisabled(value: boolean)` - управляет доступностью кнопки оформления заказа.

При клике на кнопку оформления заказа генерируется событие `order:open`.

### Класс Success

Отвечает за отображение сообщения об успешном оформлении заказа. Самостоятельный компонент, не зависящий от `Modal`.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` - принимает брокер событий для обработки кнопки закрытия и корневой DOM-элемент.

Поля класса:  
`description: HTMLElement` - элемент с описанием списанной суммы.  
`closeButton: HTMLButtonElement` - кнопка закрытия.  
`events: IEvents` - брокер событий.

Методы класса:  
`set total(value: number)` - устанавливает текст списанной суммы.

При клике на кнопку закрытия генерируется событие `modal:close`.

### Класс Card\<T>

Базовый (родительский) класс для отображения карточки товара. Карточка товара используется в трёх разных местах интерфейса, отличающихся разметкой: в каталоге, в окне предпросмотра и в списке корзины. В этот класс вынесен функционал, общий для всех трёх случаев. Самостоятельно не используется - для отображения карточек используются только дочерние классы `CardCatalog`, `CardPreview` и `CardBasket`. Отображаемые данные по составу полей соответствуют интерфейсу `IProduct`.

Сам базовый класс не работает с брокером событий и не вешает обработчики - реакцию на действия пользователя добавляют дочерние классы, принимая опциональный объект `ICardAction` с колбэком `onClick`, который определяет Презентер.

Конструктор:  
`constructor(container: HTMLElement)` - принимает корневой DOM-элемент карточки (клон содержимого темплейта).

Поля класса:  
`titleElement: HTMLElement` - элемент названия товара.  
`priceElement: HTMLElement` - элемент цены товара.

Методы класса:  
`set title(value: string)` - устанавливает название товара.  
`set price(value: number | null)` - устанавливает цену товара; если значение `null`, выводит текст «Недоступно».

### Класс CardCatalog extends Card

Отвечает за отображение карточки товара в каталоге на главной странице.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardAction)` - принимает корневой DOM-элемент карточки и опциональный объект с обработчиком клика.

Поля класса:  
`categoryElement: HTMLElement` - элемент категории товара.  
`imageElement: HTMLImageElement` - элемент изображения товара.

Методы класса:  
`set category(value: string)` - устанавливает текст категории и CSS-класс цветового модификатора, соответствующий категории.  
`set image(value: string)` - устанавливает изображение товара через утилитарный метод `setImage`.

При клике на карточку вызывается обработчик `onClick`, переданный в конструктор. Генерацию события `card:select` выполняет сам обработчик - его определяет Презентер в момент создания карточки.

### Класс CardPreview extends Card

Отвечает за подробное отображение товара, используется как самостоятельный компонент внутри модального окна.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardAction)` - принимает корневой DOM-элемент и опциональный обработчик клика по кнопке.

Поля класса:  
`categoryElement: HTMLElement` - элемент категории товара.  
`imageElement: HTMLImageElement` - элемент изображения товара.  
`descriptionElement: HTMLElement` - элемент описания товара.  
`buttonElement: HTMLButtonElement` - кнопка добавления/удаления товара из корзины.

Методы класса:  
`set category(value: string)` - аналогично `CardCatalog`.  
`set image(value: string)` - аналогично `CardCatalog`.  
`set description(value: string)` - устанавливает текст описания товара.  
`set buttonText(value: string)` - устанавливает текст кнопки. Какой именно текст («Купить» / «Удалить из корзины» / «Недоступно») - решает Презентер по цене товара и его наличию в корзине; представление само состояние не вычисляет.  
`set buttonDisabled(value: boolean)` - блокирует кнопку, если товар нельзя добавить в корзину (например, цена равна `null`).

При клике на кнопку вызывается обработчик `onClick`. Какое событие сгенерировать - `card:add` или `card:remove` - решает сам обработчик, в зависимости от текущего состояния корзины на момент клика.

### Класс CardBasket extends Card

Отвечает за отображение товара в виде строки списка корзины.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardAction)` - принимает корневой DOM-элемент строки корзины и опциональный обработчик клика по кнопке удаления.

Поля класса:  
`indexElement: HTMLElement` - элемент порядкового номера товара в списке.  
`deleteButtonElement: HTMLButtonElement` - кнопка удаления товара из корзины.

Методы класса:  
`set index(value: number)` - устанавливает порядковый номер товара в корзине.

При клике на кнопку удаления вызывается обработчик `onClick`, переданный в конструктор. Генерацию события `basket:remove` выполняет сам обработчик - его определяет Презентер в момент создания строки корзины.

### Класс Form\<T>

Базовый класс для отображения форм. Является дженериком и принимает в переменной `T` тип данных, специфичных для конкретной формы. В этот класс вынесен общий для всех форм функционал: управление доступностью кнопки отправки и отображение текста ошибок валидации. От этого класса наследуются `Order` и `Contacts`.

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` - принимает корневой элемент формы и брокер событий; вешает обработчики `submit` формы и `input` всех её полей.

Поля класса:  
`submitButtonElement: HTMLButtonElement` - кнопка отправки формы.  
`errorsElement: HTMLElement` - элемент для вывода текста ошибок валидации.  
`formName: string` - имя формы (атрибут `name` у `<form>`), служит префиксом её событий.  
`events: IEvents` - брокер событий.

Методы класса:  
`set valid(value: boolean)` - управляет атрибутом `disabled` кнопки отправки формы.  
`set errors(value: string)` - выводит текст ошибки валидации.  
`emitChange(field: string, value: string): void` - генерирует событие `${formName}:change` с данными изменённого поля. Вызывается при вводе в текстовые поля, а также дочерним `Order` для кнопок оплаты.

При вводе в поле генерируется событие `${formName}:change`, при отправке - `${formName}:submit`. Значения полей форма наружу не отдаёт; валидность и текст ошибок ей выставляет Презентер через сеттеры.

### Класс Order extends Form

Отвечает за отображение формы выбора способа оплаты и адреса доставки.

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` - вызывает конструктор родителя `Form` и вешает обработчики клика на кнопки выбора способа оплаты.

Поля класса:  
`cardButtonElement: HTMLButtonElement` - кнопка оплаты онлайн (`button[name="card"]`).  
`cashButtonElement: HTMLButtonElement` - кнопка оплаты при получении (`button[name="cash"]`).  
`addressInputElement: HTMLInputElement` - поле ввода адреса доставки (`input[name="address"]`).

Методы класса:  
`set payment(value: TPayment | '')` - выделяет модификатором `button_alt-active` активную кнопку выбранного способа оплаты и снимает выделение с другой.  
`set address(value: string)` - устанавливает значение поля адреса.

Кнопки оплаты не являются полями ввода, поэтому при клике по ним `Order` вручную вызывает `emitChange('payment', ...)` - изменение оплаты идёт тем же событием `order:change`, что и адрес.

### Класс Contacts extends Form

Отвечает за отображение формы ввода контактных данных покупателя.

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` - вызывает конструктор родителя `Form`; ввод в поля обрабатывается общим для форм обработчиком `input`.

Поля класса:  
`emailInputElement: HTMLInputElement` - поле ввода email (`input[name="email"]`).  
`phoneInputElement: HTMLInputElement` - поле ввода телефона (`input[name="phone"]`).

Методы класса:  
`set email(value: string)` - устанавливает значение поля email.  
`set phone(value: string)` - устанавливает значение поля телефона.

## События моделей данных

Модели данных следуют паттерну Observer: при изменении своего состояния они публикуют событие через брокер, не зная о подписчиках. Презентер на эти события перерисовывает представления.

| Событие           | Когда генерируется                                                   | Данные |
| ----------------- | -------------------------------------------------------------------- | ------ |
| `catalog:changed` | сохранён массив товаров (`ProductsModel.setItems`)                   | -      |
| `preview:changed` | выбран товар для подробного показа (`ProductsModel.setSelectedItem`) | -      |
| `basket:changed`  | изменён состав корзины (`addItem` / `removeItem` / `clear`)          | -      |
| `buyer:changed`   | изменены данные покупателя (`BuyerModel.setData` / `clear`)          | -      |

## Презентер

Презентер не вынесен в отдельный класс - вся логика связывания слоёв находится в основном скрипте `src/main.ts`. Приложение одностраничное, поэтому одного презентера достаточно.

Презентер подписывается на события и реагирует на них, но сам события не генерирует. События приходят из двух источников: модели данных уведомляют об изменении своего состояния (`catalog:changed`, `preview:changed`, `basket:changed`), а представления - о действиях пользователя (через брокер либо через `onClick`-обработчик, который презентер передаёт компоненту при создании; генерацию события в этом случае выполняет сам обработчик).

Поток данных следует паттерну MVP:

`представление - (событие действия) - презентер - (вызов метода) - модель - (событие изменения) - презентер - (сеттеры) - представление`

Вспомогательные функции `resolveImageUrl` (достраивает путь к изображению: исправляет расширение `.svg` - `.png` и добавляет адрес CDN) и `pluralize` (склонение существительных по числу) вынесены в `src/utils/utils.ts`.
